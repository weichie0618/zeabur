## 🔧 技術選型理由

### 為什麼選擇 shadcn/ui？

| 考量 | 說明 |
|------|------|
| **穩定性** | 組件代碼在專案中，不受套件更新影響 |
| **可控性** | 可以直接修改組件代碼 |
| **整合性** | 與 Tailwind CSS 完美配合 |
| **可訪問性** | 基於 Radix UI，符合 WCAG 標準 |
| **輕量** | 只安裝需要的組件 |

### 為什麼選擇 React Icons？

| 考量 | 說明 |
|------|------|
| **多樣性** | 包含 20+ 圖標集 (FontAwesome, Heroicons, Feather 等) |
| **按需載入** | 只打包使用的圖標 |
| **一致性** | 統一的 API |

### 為什麼選擇 React Hook Form + Zod？

| 考量 | 說明 |
|------|------|
| **性能** | 非受控表單，減少 re-render |
| **類型安全** | Zod schema 自動推導類型 |
| **驗證** | 客戶端 + 伺服器端驗證 |
